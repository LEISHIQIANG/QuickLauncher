# 命令面板独立化优化完成记录

## MVP 完成矩阵

- 第一阶段：中键弹窗只作为入口，`panel` 命令进入独立命令面板；旧弹窗结果卡片保留兼容。
- 第二阶段：命令面板支持 `text/log/table/kv/list/progress/qr` 渲染，完整 action 区，CMD/Python 捕获输出进入面板。
- 第三阶段：执行服务接管 registry 与捕获命令；支持参数输入、取消、重试和内存历史。
- 最终阶段：补齐 `confirm` 渲染、历史折叠、参数错误区、动作链 MVP 和配置入口。

## 动作链 MVP

- `ShortcutType.CHAIN` 与 `ShortcutItem.chain_steps` 已支持序列化和反序列化。
- 动作链步骤字段为 `id / shortcut_id / enabled / stop_on_error / delay_ms`。
- 第一版只引用已有快捷方式，不支持嵌套链、条件、循环、变量映射或并发。
- 执行服务通过 `run_shortcut_chain()` / `execute_shortcut_chain_sync()` 统一写入 `CommandResultStore`。
- 结果使用 `display_type="list"` 与 `window_size="large"` 展示步骤报告。

## 兼容保留项

- `popup_command_result.py` 与旧 `show_command_result()` 仍保留，作为迁移期兼容路径。
- `set_pending_command_result()` / `take_pending_command_result()` 仍保留给旧 builtin 兼容层；捕获输出与动作链主路径不依赖它。
- 命令历史仍为内存最近 50 条，不落盘。

## 后续非 MVP

- 历史落盘与命令历史搜索。
- 复杂动作链：条件、变量映射、并发、嵌套链、失败补偿。
- 真正强杀 registry handler 与独立任务管理器。
- 动作链步骤级参数输入和结构化输出传递。
