# QuickLauncher — Claude Code 项目指南

## 项目概述

QuickLauncher 是一个 Windows 桌面快速启动器。按下鼠标中键可在屏幕任意位置唤出搜索面板，支持应用、文件夹、URL、命令、快捷键、批量启动、宏录制、插件，以及中文拼音搜索和中英双语界面。

- **版本**: 1.6.1.0（唯一来源 `core/version.py:8`）
- **语言**: Python 3.12, PyQt5, C++ (原生钩子 DLL)
- **平台**: Windows 10/11
- **构建**: Nuitka + Inno Setup
- **测试**: pytest, coverage, pre-commit (black + ruff + 静默异常检查)

## 重要约定

- **Python 版本**: 采用 Python 3.12，优先在 `E:\` 盘查找已安装的 Python 3.12 解释器
- **UI 样式**: 禁止自行创建新的 UI 样式（颜色、字体、边距等），除非明确要求新建样式。新功能应复用 `ui/styles/` 中已有的样式定义

## 代码规范

### 格式化与 Lint

- **Black**: 行宽 120, 目标 Python 3.12
- **Ruff**: 行宽 120, 规则集 E/F/W/I/B/C4/UP, 忽略 E501/B905
- **mypy**: `check_untyped_defs=False`, `warn_unused_ignores=True`, `warn_return_any=False`
- **Pre-commit**: black → ruff --fix → 自定义静默异常检查

```bash
# 格式化与 lint
black .
ruff check --fix .

# 类型检查
mypy core/ ui/ hooks/ bootstrap/ services/

# 静默异常专项检查
python scripts/check_silent_exceptions.py
```

### 导入规范

- **所有 Qt 导入统一从 `qt_compat.py` 获取**（禁止直接 `from PyQt5 import ...`）
- `qt_compat.py` 提供了自动国际化翻译的包装类（QCheckBox, QLabel, QPushButton 等）和平台特有 API 的 try/except 保护（QSvgRenderer, QtWin）
- 如需新增 Qt 符号，必须先加入 `qt_compat.py` 再导入
- `core/` 不应依赖 `ui/`（架构层边界）
- `core/diagnostics.py` 例外：通过依赖注入访问 services/hooks

### 命名约定

- Qt 方法使用 camelCase（`setObjectName`, `setText`）——不适用 ruff N802 时加 `# noqa: N802`
- Python 方法/函数使用 snake_case
- 常量使用 UPPER_SNAKE_CASE

## 架构约定

### Mixin 模式

`ui/tray_app.py` 使用 Mixin 拆分为多个文件，每个 Mixin 负责一个功能域：

| Mixin 文件 | 职责 |
|-----------|------|
| `hooks_mixin.py` | 鼠标/键盘钩子生命周期 |
| `popup_mixin.py` | 弹出式搜索面板管理 |
| `sleep_mixin.py` | 系统休眠/唤醒处理 |
| `windows_mixin.py` | Windows 窗口管理 |

Mixin 不应在 `__init__` 中调用 `super().__init__()` 以外的父类方法；初始化逻辑放在 `init_mixin()` 方法中由 `tray_app.py` 显式调用。

### pyqtSignal 跨线程

- **禁止在非 UI 线程直接操作 Qt 控件**
- 后台线程统一通过 `pyqtSignal` 向 UI 线程发送结果
- 临时线程的信号连接必须在 `closeEvent` 或析构中断开，防止回调已销毁对象
- 常驻对象的信号使用 `Qt.WeakConnection` 连接

### 安全预处理管道

`core/preprocessing/` 包含命令执行前的安全检查管道，按序执行：
```
validators → sanitizers → security → rate_limiter → audit
```

每个阶段可中止执行并返回错误消息。

### 国际化

- 使用 `core/i18n.py` 的 `_("key", "default_cn")` 函数
- `qt_compat.py` 中的包装控件（QCheckBox, QLabel, QPushButton 等）自动调用 `tr()`
- 支持 `zh_CN` / `en_US` 双语；默认语言为中文
- 所有 UI 字符串应使用 `_()` 包装，禁止硬编码中文
- `_EN_US` 字典中缺少英译时，英文模式会回退显示中文

## 安全编码规范

### subprocess 安全

```python
# ✅ 正确：list 参数模式 + shell=False
subprocess.Popen(["cmd", "/c", "start", "", target, file_path], shell=False)

# ❌ 禁止：shell=True（命令注入风险）
subprocess.run(f"netsh wlan show profile name={name}", shell=True)
```

- 所有 `subprocess` 调用必须使用 list 参数模式
- 涉及用户输入的命令参数必须验证格式（如 Wi-Fi 名称用 `re.match(r'^[\w\- ]+$', ...)`）
- 优先使用 `os.startfile()` 打开文件/URL 而非 `subprocess`

### 日志与隐私

- **info 级别禁止记录用户路径、命令参数、命令内容**（PII 合规要求）
- 用户路径脱敏：`_sanitize_path(path)` 工具函数只保留 `.../parent/file`
- `debug` 级别可记录完整路径，但确保默认日志配置不输出 debug
- 使用 `logger.exception()` 记录异常（含完整 traceback），避免内联 `import traceback`

### exec/eval 禁止

```python
# ❌ 禁止：exec/eval 执行代码
exec((root / "core" / "version.py").read_text(), namespace)

# ✅ 替代：AST 解析
import ast
tree = ast.parse(text)
```

### 网络服务安全

- 所有本地服务器必须绑定 `127.0.0.1` 而非 `0.0.0.0`
- 临时文件必须在 `atexit` 或服务关闭时清理
- IPC 管道名不得暴露敏感信息

## 错误处理规范

### 日志级别

| 级别 | 场景 | 示例 |
|------|------|------|
| `logger.debug()` | 详细信息（含用户路径） | 命令执行参数 |
| `logger.info()` | 系统级事件（不含用户数据） | 服务启动/停止 |
| `logger.warning()` | 可恢复的问题 | 配置缺失使用默认值 |
| `logger.error(exc_info=True)` | 异常但可继续运行 | 文件读取失败 |
| `logger.exception()` | except 块中捕获异常 | 数据库操作失败 |
| `logger.critical()` | 严重故障 | 钩子初始化失败 |

### 静默异常禁止

```python
# ❌ 禁止：无日志的 except: pass
except: pass

# ✅ 正确：至少记录 warning
except Exception:
    logger.warning("操作失败", exc_info=True)

# ✅ 正确：特定异常 + 日志
except FileNotFoundError:
    logger.error("文件不存在", exc_info=True)

# ⚠️ 允许静默的场景（缩略为 one-liner 并添加 noqa 注释）
except Exception:  # noqa: S110 — 预期可能失败的非关键操作
    pass
```

### 异常类型精确

- 优先捕获特定异常（`FileNotFoundError`, `json.JSONDecodeError`, `ConnectionError`）
- 避免裸 `except: pass` 和过宽的 `except Exception: pass`
- 事件处理器中最低使用 `logger.error(exc_info=True)`
- 系统退出信号（`KeyboardInterrupt`, `SystemExit`）不应被吞掉

### 自定义异常

- 优先使用内置异常；仅在需要调用方特殊处理时定义自定义异常
- 自定义异常应继承 `Exception`，放在对应模块顶部

## Qt 控件生命周期管理

### 窗口关闭与释放

```python
def closeEvent(self, event):
    self.disconnect_signals()    # 断开所有信号连接
    self.save_geometry()         # 保存窗口位置/大小
    event.accept()

def close(self):
    super().close()
    self.deleteLater()  # 确保 Qt 事件循环最终释放 C++ 对象
```

- 所有窗口关闭路径必须确保 `deleteLater()` 或 `WA_DeleteOnClose` 被调用
- 隐藏窗口（如 popup）不能累积：关闭时清理旧实例
- 窗口/对话框的 `closeEvent` 中添加信号断开逻辑

### QTimer 使用

```python
# ❌ 禁止：孤立 QTimer
QTimer().singleShot(100, self.doSomething)

# ✅ 正确：指定 parent
QTimer.singleShot(100, self.doSomething)  # 静态方法安全
# 或
QTimer(self).start(100)  # 实例方法指定 parent
```

- 所有 `QTimer()` 必须指定 parent
- 单例类应复用同一个 timer 实例而非每次重建

### QThread 安全

```python
# ✅ 正确：协作取消模式
worker.quit()
worker.requestInterruption()
if not worker.wait(3000):
    worker.terminate()  # 仅超时后保底
worker.deleteLater()

# ❌ 禁止：无条件 terminate()
worker.terminate()  # 官方文档标注不安全
```

- 优先使用 `requestInterruption()` + 线程内检查 `isInterruptionRequested()`
- `terminate()` 仅作为超时（3s）后保底手段

### 信号连接管理

- 窗口/对话框的 `closeEvent` 中断开所有信号连接
- 常驻对象（tray_mixins）使用 `Qt.WeakConnection`
- lambda 回调中使用 `weakref.proxy` 避免指向已删除对象

## IPC 与原生钩子

### IPC 通信

- 使用 `QLocalServer`/`QLocalSocket` 实现单实例
- 管道名：`"QuickLauncherInstance_v3"`
- IPC 消息体**无版本字段**，服务器/客户端必须锁步

### C++ 钩子 DLL

`hooks_dll/hooks.cpp` 注意事项：
- 回调函数指针应使用 `std::atomic` 存储（防止竞态条件）
- 全局变量封装到 `HookContext` 结构体管理
- 内核句柄使用 RAII 包装（避免泄漏）
- 入口点用 `extern "C" __declspec(dllexport)`

### COM 与 Windows API

```python
# COM 初始化必须配对
import pythoncom

class ComScope:
    def __enter__(self):
        pythoncom.CoInitializeEx(pythoncom.COINIT_APARTMENTTHREADED)
        return self
    def __exit__(self, *args):
        pythoncom.CoUninitialize()
```

- 所有 `CoInitializeEx()` 调用必须有对应 `CoUninitialize()`（使用 `try/finally` 或上下文管理器）
- `keybd_event` 已弃用，改用 `SendInput`
- Windows 路径优先使用 `os.environ.get("SystemRoot", "C:\\Windows")` 动态获取
