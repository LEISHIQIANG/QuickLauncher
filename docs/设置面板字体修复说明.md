# 设置面板字体修复说明

## 修复时间
2026-02-05

## 问题描述
设置窗口的字体和配置面板的字体看起来不一样，显示效果不一致。

### 根本原因
1. **使用了 `setPixelSize()` 而不是 `setPointSize()`**
   - 像素大小（pixel）和点大小（point）的渲染方式不同
   - `setPixelSize(13)` 在不同 DPI 下显示效果不一致

2. **样式表中缺少 `font-family` 和 `font-weight`**
   - 未指定字体族，导致回退到系统默认字体
   - 使用 "bold" 而不是数值字重（600）

3. **内联样式覆盖了全局字体设置**
   - 大量内联 `setStyleSheet()` 未指定字体族
   - 字重不一致（有的用 bold，有的用数值）

## 修复内容

### 1. 移除 NavigationItem 的 setPixelSize()

**位置：** `ui/config_window/new_settings_panel.py` 第95-101行

**修复前：**
```python
# Explicitly set bold font
font = self.font()
font.setBold(True)
font.setPixelSize(13)  # ❌ 使用像素大小
self.setFont(font)
```

**修复后：**
```python
# 字体通过样式表和全局字体设置，不在这里单独设置
```

**原因：**
- `setPixelSize()` 使用绝对像素值，不随 DPI 缩放
- `setPointSize()` 使用点值，会随 DPI 正确缩放
- 最佳做法是通过样式表统一设置

### 2. 更新 NavigationWidget 样式表

**位置：** `ui/config_window/new_settings_panel.py` 第112-164行

**修复前：**
```css

    font-weight: 600;   /* ✅ 选中时稍粗 */
}
```

**改进：**
- 添加 `font-family` 确保使用思源黑体
- 使用数值字重：500（Medium）和 600（SemiBold）
- 选中项使用稍粗的字重，更醒目

## 字体设置标准化

### 字体族统一
```css
font-family: "Source Han Sans SC", "Microsoft YaHei", sans-serif;
```

### 字重规范

| 数值 | CSS名称 | 用途 | 修改前 |
|-----|---------|------|--------|
| 400 | Normal | 正文、说明文字 | 无设置或 normal |
| 500 | Medium | 按钮、导航项 | bold |
| 600 | SemiBold | 标题、强调文本 | bold |

**为什么不用 700（Bold）：**
- 思源黑体的 Regular 字重本身比微软雅黑略粗
- 600（SemiBold）在思源黑体上已经很醒目
- 700 会显得过粗，影响阅读体验

### 字体大小规范

| 大小 | 用途 | 示例 |
|-----|------|------|
| 24px | 大标题 | 关于页面标题 |
| 14px | 分组标题、副标题 | QGroupBox 标题 |
| 13px | 标准文本 | 导航、按钮、选项 |
| 12px | 说明文字 | 备份说明 |
| 11px | 辅助文字 | 小提示、描述 |

## setPixelSize vs setPointSize

### 技术对比

| 方法 | 单位 | DPI缩放 | 使用场景 |
|-----|------|---------|---------|
| `setPixelSize()` | 像素 | ❌ 不缩放 | 固定尺寸UI |
| `setPointSize()` | 点（pt） | ✅ 自动缩放 | 响应式UI（推荐） |

### 实际效果

**100% DPI (96 DPI)：**
- `setPixelSize(13)` → 13 像素
- `setPointSize(10)` → 13.3 像素

**125% DPI (120 DPI)：**
- `setPixelSize(13)` → 13 像素（未缩放）❌
- `setPointSize(10)` → 16.7 像素（正确缩放）✅

**150% DPI (144 DPI)：**
- `setPixelSize(13)` → 13 像素（未缩放）❌
- `setPointSize(10)` → 20 像素（正确缩放）✅

**结论：**
- 使用 CSS `font-size: 13px` 或 `setPointSize()`
- 避免使用 `setPixelSize()`

## CSS font-weight vs bold

### 字重对照表

| CSS 值 | Qt 枚举 | 名称 | 外观 |
|--------|---------|------|------|
| 100 | Thin | 极细 | 非常细 |
| 200 | ExtraLight | 特细 | 很细 |
| 300 | Light | 细 | 较细 |
| **400** | **Normal** | **正常** | **标准粗细** |
| **500** | **Medium** | **中等** | **略粗** |
| **600** | **SemiBold** | **半粗** | **醒目** |
| 700 | Bold | 粗体 | 粗 |
| 800 | ExtraBold | 特粗 | 很粗 |
| 900 | Black | 极粗 | 非常粗 |

### CSS "bold" 的问题

```css
/* ❌ 不推荐：不精确 */
font-weight: bold;  /* 可能映射到 700 或 600，不确定 */

/* ✅ 推荐：精确控制 */
font-weight: 600;   /* 明确使用 SemiBold */
```

**原因：**
- `bold` 是相对关键字，浏览器/框架可能解释为不同数值
- Qt 中 `bold` 通常映射到 700（Bold）
- 思源黑体 Regular + 700 显得过粗
- 使用数值（600）可以精确控制外观

## 最佳实践总结

### ✅ 推荐做法

1. **字体大小：** 使用 CSS `font-size` 或 `setPointSize()`
2. **字体族：** 始终指定 `font-family`
3. **字重：** 使用数值（400/500/600）而不是关键字（normal/bold）
4. **样式表：** 通过 CSS 统一设置，避免程序中单独设置
5. **继承：** 尽量让组件继承全局字体设置

### ❌ 避免做法

1. **不要用：** `setPixelSize()`
2. **不要用：** `font-weight: bold`
3. **不要：** 省略 `font-family`
4. **不要：** 在程序中单独设置字体（除非必要）
5. **不要：** 混用不同的字体设置方式

---

## 结论

通过移除 `setPixelSize()`、统一 `font-family` 和使用数值字重，设置面板的字体现在与配置面板完全一致，显示效果清晰统一。
