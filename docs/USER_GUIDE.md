# QuickLauncher 使用说明

QuickLauncher 是一款轻量级桌面快捷启动工具：用鼠标快速唤出启动面板，一键启动常用程序/文件/文件夹，或打开网址、执行命令、发送快捷键。

## 1. 快速上手

### 1.1 启动与托盘

启动后会常驻系统托盘（右下角通知区域）。

- 托盘图标：左键打开“设置”
- 全局唤出：在屏幕任意位置点击鼠标中键
- 退出：托盘图标右键菜单 → 退出

### 1.2 一分钟完成配置

1. 打开“设置”窗口
2. 左侧新建分类（或使用默认“常用”）
3. 在中间区域拖入文件/文件夹/快捷方式（`.lnk`）即可生成图标
4. 中键唤出启动器，点击图标启动

## 2. 启动器（弹出窗口）使用

### 2.1 唤出方式

- 默认：鼠标中键唤出（在鼠标当前位置附近弹出）
- 特殊应用：当当前前台软件命中“特殊触发（Ctrl+中键）”列表时，只有 `Ctrl + 中键` 才会唤出（用于避免 3D/CAD 软件中键冲突）

### 2.2 基本操作

- 启动：左键点击图标
- 强制新开：`Alt + 左键` 点击图标（即使已存在窗口，也尝试启动新实例）
- 关闭：`Esc`，或点击到窗口外（未锁定时）
- 锁定/常驻：在弹出窗口空白处点右键切换锁定状态；锁定后右上角会出现小圆点提示

### 2.3 翻页与 Dock

- 多分类分页：每个“分类”对应一页（Dock 除外）
- 切换分页：滚轮切换；或键盘方向键 `← / →` 切换
- Dock：顶部/底部的 Dock 区用于放置常驻图标（在设置里选择“Dock”分类编辑）

### 2.4 拖放联动（把文件拖到图标上打开）

当图标类型为“文件/文件夹”时，支持把文件拖到该图标上，实现“用某程序打开文件”或“把文件投递到某目录”：

- 拖到“程序/文件”类图标：尝试将拖入的文件作为参数交给目标程序
- 拖到“文件夹”类图标：打开目标文件夹（用于快速归档/定位）

也支持从资源管理器拖文件到弹出窗口的图标上操作；拖放过程中窗口不会自动关闭。

### 2.5 透明度快捷调节

- 调背景透明度：`Ctrl + 滚轮`
- 调图标透明度：`Shift + 滚轮`

## 3. 设置窗口使用

设置窗口分三栏：

- 左侧：分类（文件夹）列表
- 中间：当前分类的图标网格
- 右侧：全局设置（主题、透明度、尺寸、特殊触发、配置导入导出）

标题栏左上角会显示 `app.ico`（来自程序目录/项目根目录的图标）。

### 3.1 分类（左侧）

- 新建分类：点击左下角“➕ 新建分类”
- 切换分类：单击分类名
- 重命名/删除：右键分类名（系统分类如“Dock/常用”不可删除）
- 调整分类顺序：在列表中拖拽分类进行排序

### 3.2 图标网格（中间）

- 添加快捷方式：
  - 拖拽文件/文件夹/`.lnk` 到网格区域
  - 或点击底部按钮：
    - `📁 快捷方式`（程序/文件/文件夹）
    - `⌨ 快捷键`
    - `🌐 URL`
    - `⚡ 命令`
- 编辑：双击图标；或右键图标 → 编辑
- 删除：右键图标 → 删除
- 排序：拖拽图标调整顺序

`.lnk` 会尝试解析出真实目标、启动参数与工作目录（取决于系统组件可用性）。

### 3.3 右侧设置项说明

#### 启动

- 启动时显示设置窗口：勾选后每次启动都先打开设置；取消勾选后可“静默启动”（仅托盘常驻）

#### 主题

- 深色 / 浅色：影响设置窗口与启动器的整体配色风格

#### 自定义背景

- 默认背景 / 自定义图片：选择是否使用自定义图片作为启动器背景
- 选择图片：点击 `📁` 按钮
- 模糊：0–50（值越大越模糊，处理耗时也会增加）

#### 透明度

- 背景：启动器主体透明度（0–100）
- Dock：Dock 背景透明度（0–100）
- 图标：图标不透明度（10–100）

#### 尺寸

- 图标：16–48 px
- 格子：32–80 px（每个格子的占用空间，影响密度与可点击区域）
- 列数：3–10（每行显示多少列）
- 圆角：0–20 px（启动器圆角）

#### 特殊触发（Ctrl+中键）

用途：当你在某些软件中频繁用中键（旋转视图/平移/缩放等），为了避免冲突，可让这些软件里必须按 `Ctrl + 中键` 才唤出 QuickLauncher。

- 添加：点击 `＋`
- 删除：选中后点击 `－`
- 排序：拖拽列表项排序
- 编辑：双击列表项编辑
- 重置默认：恢复内置推荐列表
- 应用：立即生效并同步给鼠标钩子

#### 配置管理

- 导出配置：生成 `*.qlpack`（本质为 zip 包）
- 导入配置：从 `*.qlpack` 导入

说明：

- 导出/导入主要覆盖：`快捷键`、`URL`、`命令` 这三类项目，并尽量携带其自定义图标
- 导入时会新建一个“导入的项目 YYYY-MM-DD HH:MM”分类（或导入到指定分类）

## 4. 快捷方式类型说明

### 4.1 快捷方式（文件/文件夹/程序）

- 目标：可执行文件、任意文件、任意文件夹路径
- 参数：可选（用于传递启动参数）
- 工作目录：可选
- 图标：可选（支持选择 `ico/png/jpg/exe/dll` 等来源）

### 4.2 URL

- 网址：支持 `https://...`；未写协议时会自动补全为 `https://`
- 图标：可选（不设置则使用默认图标）

### 4.3 快捷键

- 快捷键组合：例如 `Ctrl + Shift + S`
- 触发模式：
  - 立即触发：点击后立刻执行
  - 窗口关闭后触发：等待启动器关闭后再发送按键（适合需要恢复焦点到前台窗口的场景）

### 4.4 命令

- CMD 命令：静默执行（不弹出黑窗口）
- Python 代码：执行一段 Python 代码（适合高级用户）
- 内置命令（示例）：
  - 置顶/取消置顶（Toggle Topmost）
  - 打开控制面板 / 此电脑 / 回收站

## 5. 注意事项与排障

### 5.1 关于“中键无效/不触发”

- 确认程序是否正在运行：托盘是否有图标
- 如果你正在使用“特殊应用”中的软件：请尝试 `Ctrl + 中键`
- 某些安全软件/鼠标驱动/宏工具可能会拦截全局鼠标钩子：尝试关闭冲突软件或将 QuickLauncher 加入信任

### 5.2 关于“弹窗一闪而过/自动关闭”

- 未锁定时：弹窗失焦或鼠标离开会自动关闭，属于正常行为
- 需要常驻时：在弹窗空白处右键锁定（右上角小圆点亮起）
- 拖放文件时：拖放过程中弹窗会保持不自动关闭

### 5.3 关于“图标不显示/显示异常”

- 尝试在设置里给该条目指定自定义图标（选择 `ico/png/exe/dll`）
- `.lnk` 解析与系统组件有关：如果系统组件不可用，可能无法读取到目标/图标信息

### 5.4 数据与日志位置

- 数据文件：`安装目录\config\data.json`
- 图标缓存：`安装目录\icons\`
- 运行日志：`安装目录\config\error.log`（托盘菜单 → 日志 可打开查看）

### 5.5 源码运行提示（仅开发者）

如果你不是使用已打包版本，而是直接运行源码，需要确保 Qt 与 Windows 相关依赖可用，否则部分功能（如全局钩子/`.lnk` 解析/快捷键发送）会受限。
