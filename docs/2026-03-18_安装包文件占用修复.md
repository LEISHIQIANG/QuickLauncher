# 安装包文件占用问题修复

**日期**: 2026-03-18
**问题**: 更新安装时提示 "DeleteFile failed; code 5. 拒绝访问"

---

## 问题描述

### 错误信息
```
D:\QuickLauncher\mfc140u.dll
An error occurred while trying to replace the existing file:
DeleteFile failed; code 5. 拒绝访问。
```

### 原因分析

1. **文件被占用**: `mfc140u.dll` 是 VC++ 运行时库，被正在运行的程序使用
2. **占用来源**:
   - QuickLauncher.exe 主程序
   - QuickLauncherService 系统服务

3. **触发场景**:
   - 用户在程序运行时执行更新安装
   - 服务自启动后未停止就安装

---

## 解决方案

### 原有配置

安装包已有部分处理：
```ini
[Setup]
CloseApplications=yes
CloseApplicationsFilter=*.exe
```

**问题**: 只关闭 exe 程序，不处理系统服务

### 修复方案

#### 1. 添加停止服务函数

```pascal
procedure StopServiceIfRunning();
var
  ResultCode: Integer;
begin
  // 停止服务（如果正在运行）
  Exec('net', 'stop QuickLauncherService', '', SW_HIDE,
       ewWaitUntilTerminated, ResultCode);
  Sleep(500);
end;
```

#### 2. 安装前自动停止

```pascal
function InitializeSetup(): Boolean;
begin
  // 1. 停止服务
  StopServiceIfRunning();

  // 2. 主程序由 CloseApplications 自动处理

  Result := True;
end;
```

## 停止顺序

```
1. 停止系统服务 (QuickLauncherService)
   ↓
2. 关闭主程序 (QuickLauncher.exe) - 自动
   ↓
3. 替换文件
   ↓
4. 安装完成后重新启动（可选）
```

## 用户指南

### 如果仍然遇到文件占用

**手动解决步骤**:

1. 停止服务
   ```
   Win+R → services.msc
   → 找到 "QuickLauncher 快速启动服务"
   → 右键停止
   ```

2. 关闭程序
   - 右键托盘图标 → 退出
   - 或任务管理器结束进程

3. 重新运行安装包
