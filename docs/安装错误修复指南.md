# QuickLauncher 安装错误修复指南

## 问题描述

安装时出现错误：
```
E:\QuickLauncher\msvcp140.dll
An error occurred while trying to replace the existing file:
DeleteFile failed; code 5. 拒绝访问。
```

## 原因分析

1. **权限不足** - 原配置使用 `PrivilegesRequired=lowest`，无法替换系统文件
2. **程序正在运行** - QuickLauncher 进程占用文件
3. **文件被锁定** - 其他程序可能正在使用该 DLL

## 解决方案

### 方案1：重新编译安装程序（已修复）

配置文件 `scripts/installer.iss` 已更新：
- 权限要求改为 `admin`（管理员权限）
- 添加 `PrivilegesRequiredOverridesAllowed=dialog`（允许用户选择）
- 改进进程关闭过滤器

**重新编译步骤：**
```bash
# 1. 确保已安装 Inno Setup
# 2. 编译新的安装程序
iscc scripts\installer.iss
```

### 方案2：手动安装修复

如果遇到此错误，请按以下步骤操作：

#### 步骤1：关闭所有相关进程
```powershell
# 使用 PowerShell 关闭 QuickLauncher
Get-Process | Where-Object {$_.Name -like "*QuickLauncher*"} | Stop-Process -Force
```

或手动操作：
1. 按 `Ctrl+Shift+Esc` 打开任务管理器
2. 找到并结束所有 QuickLauncher 相关进程
3. 检查"详细信息"选项卡，确保没有遗漏

#### 步骤2：以管理员身份运行安装程序
1. 右键点击安装程序
2. 选择"以管理员身份运行"
3. 如果仍然失败，继续下一步

#### 步骤3：使用安全模式（如果仍然失败）
1. 重启电脑进入安全模式
2. 在安全模式下运行安装程序
3. 重启回正常模式

### 方案3：清理旧版本后重新安装

```powershell
# 1. 卸载旧版本
# 通过控制面板 -> 程序和功能 -> 卸载 QuickLauncher

# 2. 清理残留文件
Remove-Item -Path "C:\Program Files\QuickLauncher" -Recurse -Force -ErrorAction SilentlyContinue
Remove-Item -Path "$env:APPDATA\QuickLauncher" -Recurse -Force -ErrorAction SilentlyContinue

# 3. 重新安装
```

## 预防措施

1. **安装前关闭程序** - 确保 QuickLauncher 未运行
2. **使用管理员权限** - 始终以管理员身份运行安装程序
3. **关闭杀毒软件** - 临时禁用可能干扰安装的安全软件
4. **检查磁盘空间** - 确保有足够的磁盘空间

## 技术细节

### msvcp140.dll 说明
- 这是 Microsoft Visual C++ 2015-2022 运行库的一部分
- 被多个程序共享使用
- 替换时需要管理员权限

### 安装程序配置变更
```ini
; 旧配置
PrivilegesRequired=lowest

; 新配置
PrivilegesRequired=admin
PrivilegesRequiredOverridesAllowed=dialog
```

## 常见问题

**Q: 为什么需要管理员权限？**
A: 因为需要替换系统目录中的 DLL 文件，这需要提升的权限。

**Q: 可以跳过这个文件吗？**
A: 不推荐。msvcp140.dll 是程序运行的必需文件，跳过可能导致程序无法启动。

**Q: 如何确认程序已完全关闭？**
A: 使用任务管理器检查，或运行：
```powershell
Get-Process | Where-Object {$_.Name -like "*QuickLauncher*"}
```
