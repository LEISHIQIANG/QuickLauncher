# QuickLauncher 服务自启优化记录

**日期**: 2026-03-18
**优化目标**: 将开机启动速度从 15-30 秒优化到 2-5 秒，接近火绒/Clash 的启动速度

---

## 问题分析

### 原有实现的问题

1. **启动方式**: 注册表 `HKCU\Software\Microsoft\Windows\CurrentVersion\Run`
   - 启动时机：用户登录 → 桌面加载完成 → 执行 Run 键程序
   - 优先级：最低
   - 延迟：通常 15-30 秒

2. **Python 启动开销**
   - 解压运行时（50-100MB）
   - 加载依赖：PyQt6、PIL、pywin32、pynput 等
   - 初始化流程：日志清理、配置读取、单实例检查、QApplication 创建、托盘启动

3. **同步初始化**
   - 所有组件串行加载，阻塞主线程
   - 用户需等待所有初始化完成才能看到托盘图标

### 竞品对比

**火绒**:
- Windows 服务 + 原生 C++ 驱动
- 开机即加载内核驱动
- 启动时间 < 1 秒

**Clash**:
- 任务计划程序 + Go 编译
- 延迟初始化：先启动核心，UI 后加载
- 启动时间 < 3 秒

---

## 优化方案

### 方案选择：Windows 服务

**原因**:
- 启动时机最早（系统启动时加载）
- 优先级最高（系统服务级别）
- 启动速度最快（2-5 秒）

**权衡**:
- 需要管理员权限安装
- 实现复杂度较高
- 但用户体验提升最明显

## 技术实现

### 1. 核心服务实现 (core/windows_service.py)

**关键技术点**:

```python
class QuickLauncherService(win32serviceutil.ServiceFramework):
    _svc_name_ = "QuickLauncherService"
    _svc_display_name_ = "QuickLauncher 快速启动服务"
```

**启动流程**:
1. 服务在系统启动时自动运行（后台）
2. 等待用户登录（检测 explorer.exe 进程）
3. 登录后延迟 2 秒（等待桌面加载）
4. 以分离进程方式启动主程序
5. 服务继续运行，监听停止信号

**优化细节**:
- 使用 `DETACHED_PROCESS` 标志避免阻塞
- 从注册表读取安装路径，确保路径正确
- 超时机制：最多等待 60 秒用户登录

### 2. 服务管理器 (core/service_manager.py)

**功能**:
- 检查管理员权限
- 自动请求 UAC 提升权限
- 提供简单的 API：启用/禁用/检查状态

**关键代码**:
```python
def request_admin_and_enable():
    if is_admin():
        return enable_service_autostart()
    else:
        # 弹出 UAC 提示
        run_as_admin(["--install-service"])
```

### 3. 安装包集成 (scripts/installer.iss)

**修改点**:

```ini
[Tasks]
Name: "startupicon"; Description: "Start with Windows (Service Mode - Fastest)"

[Run]
Filename: "{app}\{#MyAppExeName}"; Parameters: "--install-service";
  StatusMsg: "Installing startup service...";
  Flags: runhidden waituntilterminated; Tasks: startupicon

[UninstallRun]
Filename: "{app}\{#MyAppExeName}"; Parameters: "--uninstall-service";
  Flags: runhidden waituntilterminated
```

**流程**:
- 用户勾选"开机自启" → 安装完成后自动安装服务
- 弹出 UAC 提示 → 用户授权 → 服务安装完成
- 卸载时自动清理服务

### 4. 设置界面集成 (ui/config_window/settings_panel.py)

**优化策略**:
```python
def _set_auto_start_registry(self, enable):
    if enable:
        # 优先服务方式
        success, msg = request_admin_and_enable()
        if not success and "用户取消了权限请求" in msg:
            # 弹出选择对话框
            choice = self._show_autostart_choice_dialog()
            if choice == "admin":
                # 重新请求管理员权限
                success, msg = request_admin_and_enable()
                if not success:
                    enable_auto_start()  # 降级到注册表
            elif choice == "registry":
                enable_auto_start()  # 直接使用注册表
            else:
                return  # 用户取消
        elif not success:
            enable_auto_start()  # 其他失败，降级到注册表
    else:
        # 同时禁用服务和注册表
        if is_service_enabled():
            request_admin_and_disable()
        disable_auto_start()
```

**用户体验优化**:
- 勾选"开机自启" → 弹出 UAC
- **用户取消 UAC** → 弹出选择对话框：
  - **选项1**: 以管理员重启并使用服务（最快）
  - **选项2**: 使用注册表方式（无需重启）
  - **选项3**: 取消操作
- 服务安装失败 → 自动降级到注册表方式
- 取消勾选 → 同时清理服务和注册表

**日志优化**:
- 明确标注自启方式：`开机自启动已启用（服务方式）` 或 `开机自启动已启用（注册表方式）`
- 禁用时也标注方式：`开机自启动已禁用（服务方式）` 或 `开机自启动已禁用（注册表方式）`

## Bug修复记录

### 2026-03-18 下午修复

**1. 修复选择对话框主题属性错误**
- **问题**: `'SettingsPanel' object has no attribute 'theme'`
- **原因**: SettingsPanel 使用 `self.current_theme` 而不是 `self.theme`
- **修复**: 将对话框中的 `self.theme` 改为 `self.current_theme`

**2. 修复界面闪烁问题**
- **问题**: 点击开机自启复选框时界面文字闪烁
- **原因**: `_on_auto_start_changed` 方法中 `_updating` 保护不完整
- **修复**: 使用 try-finally 确保 `_updating` 状态正确管理

**3. 修复变量名错误**
- **问题**: 用户取消时恢复复选框状态失败
- **原因**: 使用了错误的变量名 `self.auto_start_checkbox`
- **修复**: 改为正确的变量名 `self.auto_start_cb`

**4. 优化日志输出**
- **问题**: 日志不明确显示使用的自启方式
- **修复**: 所有日志明确标注"（服务方式）"或"（注册表方式）"

**5. 修复选择对话框 API 错误**
- **问题**: `'ThemedMessageBox' object has no attribute 'setText'`
- **原因**: ThemedMessageBox 构造函数直接接收参数，不支持 setText
- **修复**: 改用标准 QMessageBox，支持 setText 和 setIcon

**6. 彻底修复界面闪烁**
- **问题**: 开关"开机自启动"时文字仍然闪动
- **原因**: `settings_changed.emit()` 触发 `_on_settings_panel_changed`，导致 `_apply_theme()` 和数据重载
- **修复**:
  - 使用 `blockSignals(True)` 阻止复选框信号
  - 移除自启动处理中的 `settings_changed.emit()`，避免不必要的界面刷新

**7. 添加管理员重启功能**
- **功能**: 用户选择"以管理员重启"后，程序会以管理员权限重新启动
- **实现**: 使用 `ShellExecuteW` 以 runas 方式启动程序，并传递 `--install-service` 参数

**8. 优化选择对话框**
- **改进**: 创建自定义 QDialog 替代标准 QMessageBox
- **样式**: 清晰的按钮布局，更好的用户体验

**9. 修复管理员重启功能**
- **问题**: 点击"重启"后程序只关闭，没有重新启动
- **原因**:
  - 打包工具未设置 `sys.frozen` 标志
  - `sys.executable` 返回 `python.exe` 而不是 `QuickLauncher.exe`
  - 单实例检测导致新进程立即退出
- **修复**:
  - 直接查找 `QuickLauncher.exe` 文件
  - 支持三种模式：打包模式（查找QuickLauncher.exe）、开发模式1（同目录main.py）、开发模式2（向上查找main.py）
  - 延迟2秒关闭旧进程，避免单实例冲突
  - 添加详细的错误码映射和日志

**10. 修复服务安装错误**
- **问题**: `'QuickLauncherService' has no attribute '_svc_reg_class_'`
- **原因**: `win32serviceutil.InstallService` 第一个参数应该是类本身，不是 `_svc_reg_class_` 属性
- **修复**: 改为直接传递 `QuickLauncherService` 类

**11. 优化对话框文案和布局**
- **标题**: "开机自启动"
- **内容**: "检测到当前为非管理员身份运行，如需开启服务自启动，需要管理员身份运行，若不需要则默认采用注册表自启动，但速度会慢些。"
- **按钮**: "重启" 和 "忽略"，横向排列
