## 修复"设置"启动失败（两轮修复）

**问题**：用户执行 Dock 中名为"设置"的快捷方式（command="show_config_window", command_type="builtin"），弹出错误"启动失败 - 无法启动：设置 - 原因：内置命令执行失败"。

**根因分析**：
1. **表面修复**（第一轮）：`BUILTIN_COMMAND_ALIASES` 缺少 `"设置"` → `"show_config_window"` 映射
2. **真正根因**（第二轮）：`popup_item_execution.py` Phase 2 代码将所有非面板命令的 command 替换为 registry id（"show_config_window" → "config"），导致 `_execute_builtin_command("config")` 不认识 "config"，最终返回 False

**修复**：
1. `core/builtin_commands.py` L106：添加 `"设置": "show_config_window"`
2. `core/slash_commands.py` L42：在 config 命令别名中加入 `"设置"`
3. **`ui/launcher_popup/popup_item_execution.py` L231-250**：当 cmd_def.handler 是 `_CallbackHandler` 时，不再用 registry id 替换 command，保持原始回调名（如 "show_config_window"）以确保旧版内置分派能正确识别

## 恢复 ThemedToolWindow 首显预处理三件套

**问题**：`themed_tool_window.py` 丢失了 `prepare_first_show`、`WA_DontShowOnScreen`、`showEvent` 同步 blur；`windows_mixin.py` 丢失了 `presentation_ready` 信号连接和 `first_show` 语义；`window_effect.py` 的 `paint_win10_rounded_surface` 未使用 `make_cosmetic_pen`。

**修复**（3 个文件）：

1. **`ui/themed_tool_window.py`**：
   - 添加 `presentation_ready = pyqtSignal()` 类信号
   - `__init__` 中设置 `WA_DontShowOnScreen = True`
   - 添加 `prepare_first_show()` 方法：强制 `winId()` 创建 HWND、预应用 blur、清除 `WA_DontShowOnScreen`
   - `showEvent` 改为同步调用 `_apply_blur_effect()`（替换原 `QTimer.singleShot(10, ...)` 延迟），添加 `WA_DontShowOnScreen` 安全清除（含 Win32 `ShowWindow` 兜底），末尾 `emit presentation_ready`

2. **`ui/tray_mixins/windows_mixin.py`**：
   - `show_command_panel` 用 `first_show` 替换 `created_window` 语义
   - 首显时调用 `prepare_first_show()`，连接 `presentation_ready` → `_present_command_panel_window`
   - 运行中命令延迟 `show()` 到 `presentation_ready` 回调；注册 `QTimer.singleShot(100ms, _activate_command_panel_window)` 作为激活兜底
   - 拆分 `_present_command_panel_window`（show+activate）和 `_activate_command_panel_window`（activate only）

3. **`ui/utils/window_effect.py`**：
   - `paint_win10_rounded_surface` 内 `QPen(border, 1.0)` + 手动 `setJoinStyle`/`setCapStyle` → `make_cosmetic_pen(border, 1.0)`（自动处理 cosmetic + RoundJoin/RoundCap）

## 修复设置窗口内置命令图标显示异常

**问题**：设置窗口（ConfigWindow）中"内置命令管理"和"收藏命令"列表的图标显示奇怪。

**根因**：
1. `ui/command_icon_renderer.py` 中矢量图标渲染使用了 `make_cosmetic_pen(foreground, 6)`。`make_cosmetic_pen` 在 pixel_snap 实验特性开启时会设置 `setCosmetic(True)`，导致画笔宽度固定为 6 物理像素——无论图标缩放因子如何。在小尺寸（~22px）图标上，6px 的笔画宽度占比近 30%，严重破坏了视觉效果。
2. 图标渲染逻辑尺寸为 `sp(22)` 但所在 QLabel 为 `sp(24)`，图标内容区域仅占约 60%，视觉上偏小。

**修复**（2 个文件）：
1. **`ui/command_icon_renderer.py`**：
   - 移除 `from ui.utils.pixel_snap import make_cosmetic_pen`
   - 导入改为 `from qt_compat import ..., QPen`
   - `render_builtin_command_icon` 中：`make_cosmetic_pen(foreground, 6)` → `QPen(foreground)` + `pen.setWidthF(6.0)`
   - `_restore_stroke` 中：`make_cosmetic_pen(color, width)` → `QPen(color)` + `pen.setWidthF(width)`，参数类型从 `int` 改为 `float`
   - 核心思路：图标渲染器使用 `painter.scale(physical_size/100, ...)` 处理 DPI 缩放，画笔宽度应在变换后的坐标空间中生效（`setWidthF`），而非固定物理像素（cosmetic pen），这才是利用 100x100 虚拟坐标系随尺寸自动缩放的正确做法。

2. **`ui/config_window/settings_commands_page.py`**：
   - 收藏命令图标：`sp(22)` → `sp(24)`
   - 内置命令图标：`sp(22)` → `sp(24)`
   - 图标渲染尺寸与 QLabel 尺寸一致，消除多余 padding。
